# YMB HTML差分同期ツール

ローカルに保存しているHTMLフォルダと、公開サーバー上のHTMLとの差分を確認し、
**サーバー側の最新版でローカルファイルを上書き同期する**ツールです。

姉妹アプリ「YMB EJS差分同期ツール」(公開HTMLの差分をEJSソースへ書き戻すツール)とは逆方向で、
EJSのようなテンプレート処理は一切行わず、HTMLファイルをそのまま比較・同期します。

## できること

- ローカルHTMLフォルダ配下の`*.html`を再帰的に走査し、公開サーバー上の対応ページと内容を比較
- 差分は追加=緑・削除=赤(取り消し線)の文字単位ハイライトで表示
- 差分のあるページを1件ずつ、または一括でサーバー版に上書き同期(上書き前に自動で`.bak`バックアップを作成)
- 同期後に問題があれば、バックアップからワンクリックで復元
- `sitemap.xml`を参照し、ローカルにまだ存在しない新規ページをサーバー側から検出してローカルに保存
- Basic認証がかかった環境にも対応(ユーザー名・パスワードを指定。パスワードは保存されません)

## 使い方(開発モードで起動)

```bash
npm install
npm start
```

起動したら、

1. 「ローカルHTMLルート」でローカルのHTMLフォルダを選択します(一度入力した値は記憶され、次回起動時に自動で復元されます)
2. 「公開サーバーURL」に比較対象の公開サイトのベースURLを入力します(例: `https://www.example.com/site`)
3. Basic認証がかかっている場合はユーザー名・パスワードを入力します(パスワードは保存されません)
4. 対象サブディレクトリを指定(空欄で全体)
5. 「スキャン実行」を押すと、ローカルの各HTMLファイルに対応する公開サーバー上のURLを取得し、内容を比較します。あわせて`sitemap.xml`を確認し、ローカルにない新規ページを検出します(`sitemap.xml`が無いサイトの場合はこの検出はスキップされ、その旨がステータスに表示されます)
6. ページ一覧から確認したいページを選択します。一覧の幅はドラッグで調整でき、幅は記憶されます(ダブルクリックで既定幅にリセット)
7. 差分のあるページは「このページをサーバー版で上書き」、新規ページは「ローカルに保存」から個別に反映できます。ツールバーの「差分ページ全部をサーバー版で上書き」で一括同期もできます(「新規ページも取り込む」チェックで新規ページも対象に含められます)
8. 上書き時には対象ファイルの`.bak`バックアップが初回のみ自動作成されます。同期後に問題があれば、詳細パネルの「バックアップ(.bak)から復元」でワンクリックで元に戻せます

## 技術スタック

| 用途 | ライブラリ/ツール |
|---|---|
| フレームワーク | [Electron](https://www.electronjs.org/) v42 |
| 差分計算 | 自前実装(LCSベース、追加ライブラリ不要) |
| ビルドツール | [electron-builder](https://www.electron.build/) v26 |
| CI/CD | GitHub Actions |

## インストーラーのビルド

```bash
npm run build-icons   # アイコン生成(初回・アイコン変更時)
npm run dist:win       # Windows向け(nsisインストーラー)
npm run dist:mac       # macOS向け(dmg)
```

## ダウンロード

[Releases](https://github.com/yumebi/ymb_html_synch/releases) からインストーラーをダウンロードしてください。

> **インストール時に警告が表示される場合があります。** このアプリは個人配布用で、コード署名(証明書による発行者署名)を行っていません。
> これは悪意のあるソフトという意味ではなく、署名コストをかけていない未署名アプリに対する標準的な警告です。
> - **Windows**: 「Windows によって PC が保護されました」(Microsoft Defender SmartScreen)と表示された場合、「詳細情報」→「実行」を選択してください。
> - **macOS**: 「開発元が未確認のため開けません」と表示された場合、Finderでアプリを右クリック(またはControl+クリック)→「開く」を選択してください。初回のみこの操作が必要です。
